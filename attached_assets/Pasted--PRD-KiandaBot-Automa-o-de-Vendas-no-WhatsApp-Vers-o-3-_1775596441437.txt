## 1. Visão Geral

### 1.1 O que é o KiandaBot
O KiandaBot é um assistente automático para WhatsApp que ajuda **vendedores, donos de grupos e afiliados** a divulgar produtos, automatizar postagens, gerar links rastreados e calcular comissões de forma justa. É uma plataforma B2B/B2C que transforma grupos de WhatsApp em canais de venda organizados e rentáveis.

### 1.2 Porquê este projeto
- **Problema real:** Milhares de angolanos usam WhatsApp para vender, mas perdem horas a postar manualmente; anúncios se perdem; comissões não são rastreadas; donos de grupos não monetizam.
- **Oportunidade de mercado:** O WhatsApp é a principal ferramenta de vendas em Angola, mas não existem ferramentas de automação acessíveis que integrem vendedores, afiliados e donos de grupos.
- **Alinhamento com o Unitel Go Challenge:** Startup digital com modelo de negócio claro, escalável, impacto social e potencial de parceria com a Unitel.

### 1.3 Público‑alvo (Expandido)
| Segmento | Exemplo | Benefício para o utilizador |
|----------|---------|----------------------------|
| **Vendedores individuais** | Quem vende roupas, eletrónica, alimentos | Automatiza postagens, rastreia vendas, ganha tempo. |
| **Donos de grupos de vendas** | Administradores de grupos com centenas de membros | Ganham comissão por cada venda originada no grupo. |
| **Afiliados / divulgadores** | Pessoas que partilham produtos em vários grupos | Recebem comissão sem ter stock. O bot rastreia o link partilhado. |
| **Pequenas lojas físicas** | Lojas de bairro, salões, oficinas | Divulgam promoções e recebem pedidos via WhatsApp. |
| **Fornecedores / atacadistas** | Quem vende para revendedores | Comunicam novidades e recebem encomendas de forma automatizada. |

## 2. Estado Técnico Atual (Abril 2026)

| Funcionalidade | Estado | Detalhe |
|----------------|--------|---------|
| **Publicação em grupo de teste** | ✅ Funcional | O bot consegue postar num grupo específico. |
| **Publicação em múltiplos grupos cadastrados** | ❌ Não funcional | Suspeita de problema na API Green (limitação ou configuração). |
| **Geração de links rastreados** | ⚠️ Parcial | Funciona em simulação, mas não integrado 100% com o fluxo de postagem. |
| **Cálculo de comissões** | ⚠️ Parcial | Lógica implementada, não testada com vendas reais. |
| **API utilizada** | Green API | Gratuita, mas com limitações. Será substituída. |
| **MVP funcional para demonstração** | ✅ Sim | É possível mostrar o fluxo básico (postagem num grupo, link, comissão simulada). |

## 4. Diferenciais Competitivos (O que torna o KiandaBot único)

### 4.1 Já implementados (ou em simulação)
- **Automação + rastreamento de comissões** – base do modelo de negócio.
- **Suporte a afiliados** – qualquer pessoa pode divulgar e ganhar.
- **Modelo de receita claro** – assinatura + comissão.

### 4.2 Planeados (diferenciais fortes para o pitch)
| Diferencial | Descrição | Impacto |
|-------------|-----------|---------|
| **Sistema de reputação do vendedor** | Comprador avalia o vendedor (1–5 estrelas) após a venda. | Aumenta confiança; diferencia de bots genéricos. |
| **Relatório de melhor horário para postar** | Bot analisa histórico de cliques e sugere horários ideais. | Aumenta conversão; valor adicional. |
| **Catálogo inteligente com gestão de stock** | Vendedor mantém produtos no bot; stock atualiza automaticamente. | Elimina retrabalho; evita vender produtos em falta. |
| **Rastreamento por QR Code (offline)** | Gera QR code para ser escaneado na loja física, rastreando venda offline. | Capta vendas presenciais; inovação. |
| **Dashboard de dados para a Unitel** | Relatórios anónimos de vendas por região, horários, produtos. | Transforma o bot numa fonte de inteligência de mercado. |
| **Ranking de afiliados** | Placar público com os que mais venderam; prémios simbólicos. | Gamificação, aumenta engajamento. |

### 4.3 Visão de ecossistema (KiandaPay)
- Integração com pagamentos (Unitel Money) e logística (código de entrega).  
- Mostra que o projeto não é uma ferramenta isolada, mas uma **plataforma completa**.

## 5. Modelo de Negócio (Atualizado)

| Fonte de receita | Valor | Observação |
|------------------|-------|------------|
| **Assinatura mensal (vendedor)** | 3.000 Kz | Inclui até 50 posts/mês |
| **Comissão por venda** | 5% sobre o valor | Retida automaticamente pelo bot |
| **Comissão para dono de grupo** | 2,5% | Incentivo para adesão |
| **Comissão para afiliado** | 2,5% | Pode ser partilhada com o grupo ou ser adicional |
| **Planos premium** | 5.000 Kz/mês | Posts ilimitados, suporte prioritário |
| **Planos para lojas** | 10.000 Kz/mês | Catálogo + stock + múltiplos vendedores |

**Projeção de break‑even:** ~50 vendedores pagantes cobrem custos de API e servidor.

## 6. Arquitetura Técnica e Desafios

### 6.1 Componentes planeados
| Componente | Tecnologia | Estado |
|------------|------------|--------|
| **Bot WhatsApp** | Python + Evolution API (migração) | Em desenvolvimento |
| **Servidor** | Python (FastAPI) + SQLite | Esboçado |
| **Rastreamento** | Encurtador personalizado (ou Bitly) | Em simulação |
| **Painel web** | React / Vue (futuro) | Não iniciado |
| **Base de dados** | PostgreSQL (futuro) | Planeado |

### 6.2 Desafios técnicos atuais
| Desafio | Causa provável | Solução |
|---------|----------------|---------|
| Não posta em múltiplos grupos | API Green pode ter limitação de grupos | Migrar para **Evolution API** (mais robusta) |
| Rastreamento incompleto | Falta integração entre bot e sistema de links | Implementar middleware que gera link após postagem |
| Risco de bloqueio do WhatsApp | Uso de API não oficial | Migrar para API oficial do WhatsApp Business (após parceria) |

## 8. Estratégia para o Unitel Go Challenge

### 8.1 Foco da candidatura (primeira fase)
- **Provar conceito, não produto final.** Mostrar fluxo funcional (postagem num grupo, link, comissão simulada).
- **Destacar diferenciais** mesmo que em simulação (mockups).
- **Mostrar público‑alvo expandido** (vendedores, donos de grupo, afiliados, lojas).
- **Ligar o projeto à Unitel:** dashboard de dados, potencial integração com Unitel Money.

### 8.2 Mensagem‑chave para o pitch
> *“O KiandaBot não é só um bot de automação. É um ecossistema que organiza vendas no WhatsApp, recompensa donos de grupos e afiliados, e gera dados valiosos para a Unitel.”*

### 8.3 Riscos e mitigação
| Risco | Mitigação |
|-------|-----------|

## 10. Decisões de Design e Trade‑offs

| Decisão | Motivo |
|---------|--------|
| Usar API Green inicialmente | Gratuita, permitiu testar conceito. Agora é hora de migrar. |
| Priorizar funcionalidade de um grupo | Para ter MVP rápido. Agora, escalar é urgente. |
| Simular rastreamento e diferenciais | Para não atrasar o pitch; o importante é mostrar a visão. |
| Não integrar pagamentos ainda | Simplifica o MVP; será feito após validação do modelo. |
